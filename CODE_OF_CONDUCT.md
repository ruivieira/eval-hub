# Code of Conduct

## Our Standards

We expect all participants in the Eval Hub project to maintain professional conduct and treat others with respect.

### Expected Behavior

- **Professional Communication**: Use respectful and constructive language in all interactions
- **Constructive Feedback**: Provide helpful, actionable feedback on code and documentation
- **Collaborative Approach**: Work together to solve problems and improve the project
- **Technical Focus**: Keep discussions focused on technical matters and project goals
- **Knowledge Sharing**: Help others learn and contribute to the project
- **Responsibility**: Take ownership of your contributions and acknowledge mistakes

### Technical Standards

- **Quality Work**: Submit well-tested, documented code that follows project standards
- **Clear Communication**: Explain technical concepts clearly and ask questions when needed
- **Honest Attribution**: Give proper credit for others' work and contributions
- **Security Mindset**: Do not introduce vulnerabilities or malicious code intentionally

## Unacceptable Behavior

The following behaviors are not acceptable:

### Professional Misconduct
- **Harassment**: Unwelcome attention, stalking, or intimidation of any kind
- **Personal Attacks**: Insults, derogatory comments, or ad hominem arguments
- **Disruptive Behavior**: Deliberately derailing discussions or project activities
- **Privacy Violations**: Publishing private information without explicit permission

### Technical Misconduct
- **Malicious Code**: Submitting harmful code or introducing vulnerabilities intentionally
- **Plagiarism**: Claiming others' work as your own without proper attribution
- **False Information**: Deliberately providing misleading information or refusing constructive feedback

### Spam and Off-topic Content
- **Spamming**: Excessive self-promotion or irrelevant content
- **Off-topic**: Deliberately posting content unrelated to project goals

## Scope

This Code of Conduct applies to:
- All project repositories and communications
- GitHub issues, pull requests, and discussions
- Project documentation and wiki
- Public representation of the project

## Reporting

To report violations:
- Create a private GitHub issue for maintainers
- Contact project maintainers directly (see project metadata for current contacts)

Reports will be handled promptly and confidentially.
